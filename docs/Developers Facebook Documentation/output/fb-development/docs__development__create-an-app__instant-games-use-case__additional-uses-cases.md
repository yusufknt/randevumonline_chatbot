# Additional Launch a Game on Facebook Use Cases

This document shows you how to add more use cases to your Launch a game on Facebook use case.

Once you have implemented the Launch a game on Facebook use case on this app, you can add more use cases to give your app the functionality you want.

We recommend completing each use case before adding another one. This will allow you to view the functionality each use case provides so you don't add a use case that your app will not need.

## Available use cases

The following use cases can be added to the Launch a game on Facebook use case. You will configure most of the products listed on the app Dashboard then implement code to use them by following the instructions in the developer documentation.

| Use Case | Available permissions and Products to implement |
| --- | --- |
| **Help players find friends so they can play together** – Get more players for your game by helping current players find their friends so they can play together. | - [Sharing for Instant Games](https://developers.facebook.com/docs/games/acquire/sharing/instant-games/) - `user_friends` |
| **Create competition with a tournament** – Get new players and keep players engaged by creating competition in your game. | - [Instant Tournaments](https://developers.facebook.com/docs/games/acquire/instant-tournaments/) |
| **Build a player leaderboard** – Let players see how their gameplay compares to other players. | - [Leaderboards](https://developers.facebook.com/docs/games/acquire/leaderboards/) |
| **Share gameplay on Facebook and Instagram** – Players can share screenshots and video clips of their gameplay to Instagram and Facebook, including Stories and Facebook Groups. | - [Sharing for Gaming](https://developers.facebook.com/docs/games/acquire/sharing/native-apps) |
| **Connect with players on Messenger** – Add a game bot to Messenger to send reminders, tips, and more to enhance player experience. | - [Game Updates via Messenger](https://developers.facebook.com/docs/games/engage/game-bots/) - `pages_messaging` |
| **Show ads in your game** – Monetize your game and grow your revenue using Facebook ads. You'll need Audience Network, Ads Manager and Monetization Manager to manage ads. | - [In-App Ads for Instant Games](https://developers.facebook.com/docs/games/monetize/in-app-ads/instant-games/) - [Meta Audience Network](https://developers.facebook.com/docs/audience-network/) - [Meta Monetization Manager](https://www.facebook.com/business/help/103628146695524) |
| **Track engagement with Meta App Events** – Choose from standard app events or create your own to get data and analyses about how users interact with your app. | - [Meta App Events](https://developers.facebook.com/docs/development/create-an-app/instant-games-use-case/docs/app-events/overview/) |
| **Take in-app payments from players** – Allow players to make in-app purchases. Set up your payout information, connect a catalog and more. | - [In-App Purchases for Instant Games](https://developers.facebook.com/docs/games/monetize/in-app-purchases/instant-games/) |
| **Create a custom landing page for clicks from ads and shared media** – Create custom landing page for players that find your game through click-to-play ads, or posts and stories shared by current players. Learn more | - [Sharing for Instant Games - Custom fb.gg/play Deeplinks](https://developers.facebook.com/docs/games/acquire/click-to-play-ads#custom-payloads) - [Click-to-Play Ads - Custom Payloads](https://developers.facebook.com/docs/development/create-an-app/instant-games-use-case/docs/games/acquire/click-to-play-ads#custom-payloads) |
| **Measure return on ad spend (In development)** – Measure the revenue earned for each dollar you spend on advertising. | - [ROAS Measurement for Instant and Cloud Games (In development)](https://developers.facebook.com/docs/games/monetize/roas/) |
| **Get real-time notifications with Webhooks** – Get automatic HTML notifications when app users make changes related to the permissions that you've added to your app. | - [Meta Webhooks](https://developers.facebook.com/docs/graph-api/webhooks/) |

## Next steps
